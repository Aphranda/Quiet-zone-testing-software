本文档只跟踪 UI 实现待办，不讨论公式、不讨论校准业务模型本身。UI 目标是为普通操作人员提供一套基于 PySide6 + MVVM 的步骤式校准界面，并提供独立的指令测试页、日志页和结果页。

## 待办规则

- 任务编号使用 `UI-01`、`UI-02` 等稳定编号，不复用编号。
- 每个任务都必须有明确验收条件。
- UI 不直接拼接业务命令，数据应来自 ViewModel / Runner。
- UI 不负责公式计算和文件命名。
- 所有步骤说明、接线提示和路径摘要应与 `CATR_CHAMBER_CALIBRATION_TEST_PLAN.html` 保持一致。

## 优先级说明

| 优先级 | 目标 |
|---|---|
| P0 | UI 骨架与入口可运行。 |
| P1 | 校准执行页可显示步骤、接线路径、命令与状态。 |
| P2 | 指令测试页、日志页、结果页可用。 |
| P3 | UI 与状态机联动，支持一步一步执行 Mock 校准。 |
