更新时间：2026-07-19

本文档只跟踪 UI 实现待办，不讨论公式、不讨论校准业务模型本身。UI 目标是为普通操作人员提供一套基于 PySide6 + MVVM 的步骤式校准界面：第一页完成仪表配置，中间执行校准，结果页查看输出，右侧 LOG 始终可见。

## 待办规则

- 任务编号使用 `UI-01`、`UI-02` 等稳定编号，不复用编号。
- 每个任务都必须有明确验收条件。
- UI 不直接拼接业务命令，数据应来自 ViewModel / Runner / adapter。
- UI 不负责公式计算和文件命名。
- 所有步骤说明、接线提示和路径摘要应与 `CATR_CHAMBER_CALIBRATION_TEST_PLAN.html` 保持一致。

## 优先级说明

| 优先级 | 目标 |
|---|---|
| P0 | UI 骨架与入口可运行。 |
| P1 | 仪表配置页可完成四设备配置、连接和命令验证。 |
| P2 | 校准执行页可显示步骤、接线路径、命令与状态。 |
| P3 | 结果页、右侧 LOG 和状态机联动可用。 |
| P4 | 现场体验、异常处理和真实硬件接入完善。 |

### P3 验收

- 操作人员在任意页签都能看到 LOG。
- 运行结束后，结果页能直接看到会话级概要。
- 结果页能明确区分 raw、metadata 和 final。
