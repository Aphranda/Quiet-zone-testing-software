## TODO 规则

- 任务按 P0/P1/P2/P3 优先级分组。
- 任务编号使用 `HR-P0-01`、`HR-P1-01` 格式，编号稳定且不复用。
- 每条任务必须包含明确验收标准。
- 完成任务时只把 `[ ]` 改为 `[x]`；详细过程写入进度记录。
- 涉及真实运动或射频输出的验证必须由环境变量显式启用，默认测试不得操作硬件。
- 每阶段保持应用可启动，已有 CSV、metadata 和 trace index 格式不得无意回归。

## 验收标准摘要

| 优先级 | 验收标准 |
|---|---|
| P0 | 扫描架不会无限等待；扫描进度只代表已保存数据；目标与实际位置可追溯。 |
| P1 | 暂停、停止、通信故障和保存失败有确定状态；固定延时依赖显著减少。 |
| P2 | E5080B/N5245B、扫描架和开关箱完成真实硬件闭环验证及故障恢复验证。 |
| P3 | 数据质量、恢复续扫、可观测性和长期稳定性具备明确标准。 |

## P0 - 运动超时、进度和位置事实

- [x] HR-P0-01 修复扫描架绝对运动超时：`IclPositionerController.wait_axes()` 必须使用 `timeout_ms`，超时后停止相关轴并抛出包含轴号、目标位置、最后实际位置和等待时长的异常。
- [ ] HR-P0-02 为匀速运动位置轮询增加截止时间：目标距离和速度用于计算合理上限，同时检测位置反馈长时间不变化；超时或反馈冻结时立即停轴并报错。
- [ ] HR-P0-03 修正步进扫描完成进度：只有网分采样成功且 trace 文件及索引写入成功后，点位才标记为“已完成”；到点、稳定、采样中可使用独立阶段状态，不得占用完成计数。
- [ ] HR-P0-04 建立扫描点目标/实际位置事实：每个点保存逻辑目标、物理目标、采样前实际位置和位置误差；CSV metadata、trace index 和扫描结果模型使用同一事实来源。
- [ ] HR-P0-05 增加步进到位误差门限：采样前查询实际位置，超出轴容差时重试定位或终止扫描，禁止以目标坐标代替未验证的实际坐标。
- [ ] HR-P0-06 明确扫描坐标语义：产品和 UI 明确选择“相对启动位置”或“绝对机械坐标”；预览、动画、文件名和执行器使用相同定义，并覆盖非零起点测试。

### P0 验收

- 位置反馈冻结或永不到位时，扫描在有限时间内失败并停轴。
- 采样或保存失败的点不会显示为已完成。
- 每条 trace 可追溯目标位置、实际位置和误差。
- 坐标模式在 UI、日志、文件和实际运动中一致。
- 新增运动超时、反馈冻结、位置超差、采样失败和保存失败单元测试。

## P1 - 停止、暂停、通信和开环延时治理

- [ ] HR-P1-01 串行化扫描架命令访问：运动、位置查询、暂停和紧急停止不得并发破坏同一 Modbus 会话；停止命令具有明确优先级且不会被后续运动命令覆盖。
- [ ] HR-P1-02 定义暂停安全点：分别规定运动中、到点稳定等待中、网分采样中和文件保存中的暂停行为；UI 文案与实际可中断边界一致。
- [ ] HR-P1-03 区分用户停止与设备故障：停止请求不能吞掉同时发生的通信、运动或存储异常；结果状态至少区分 Completed、Stopped、Failed 和 StoppedWithError。
- [ ] HR-P1-04 改进 Modbus RTU 帧读取：移除依赖固定 20 ms 的一次性读取，按功能码和长度字段持续读取到完整帧或超时，并保留 CRC、地址和功能码校验。
- [ ] HR-P1-05 评估到点稳定条件：保留可配置 `settle_delay_s` 作为机械稳定保护，但记录位置稳定样本、最后运动时间和实际等待时间；验证后确定各速度/负载的默认值。
- [ ] HR-P1-06 收紧开关箱重试语义：查询或命令超时后避免无条件重复非幂等操作；重试前清理输入缓冲并确认设备协议允许重发。
- [ ] HR-P1-07 建立扫描资源仲裁：扫描期间禁止手动运动、重复采样、断开仪器或修改关键连接参数；停止清理完成前不能启动下一轮扫描。

### P1 验收

- 暂停和停止在所有扫描阶段都有自动化测试。
- 用户停止与设备异常在 UI、日志和扫描会话状态中可区分。
- Modbus 慢响应和分片响应不会依赖固定休眠碰运气。
- 同一硬件会话不存在未保护的并发读写。
- `settle_delay_s` 的默认值有真实硬件验证记录，不再只是经验常量。

## P2 - 真实硬件闭环验证

- [ ] HR-P2-01 验证 E5080B 测量通道流程：覆盖已有/不存在 `CH1_SPARAM`、S11/S21/S12/S22 切换、重复配置、错误队列和断线重连。
- [ ] HR-P2-02 验证 N5245B 测量通道流程：覆盖型号识别、测量创建/选择、单次触发、复数读取及与 E5080B 的命令差异。
- [ ] HR-P2-03 验证网分确定性触发：确认 `INIT1:CONT OFF → TRIG:SOUR IMM → INIT1:IMM → *OPC?` 每次只产生一条完整 trace，超时配置覆盖最慢合法扫描。
- [ ] HR-P2-04 验证网分数据完整性：检查 `REAL,64 + SWAP + SDATA` 的端序、点数、复数顺序、实际频率轴、幅相计算和重复采样一致性。
- [ ] HR-P2-05 验证扫描架闭环运动：覆盖正反向、长短距离、两个轴、暂停、停止、限位、反馈冻结、通信中断和到位误差。
- [ ] HR-P2-06 验证开关箱命令闭环：确认 LCD74000F/TC500 的成功响应、错误响应、超时、重发和实际射频路径一致。
- [ ] HR-P2-07 执行完整步进扫描验收：逐点核对目标位置、实际位置、网分 trace、文件名、CSV、metadata、trace index、进度和最终报告数量一致。
- [ ] HR-P2-08 执行完整匀速扫描验收：确认运动过程中采样点位置定义、越点判断、转向停轴、暂停恢复和最终停机行为符合产品要求。

### P2 验收

- 验证记录包含仪器型号、序列号、固件、选件、连接方式和测试日期。
- E5080B 与 N5245B 各至少完成一次全流程步进测试。
- 所有注入故障均有确定的超时、停机、日志和恢复结果。
- 硬件集成测试继续默认跳过，仅在 `RUN_HARDWARE_INTEGRATION=1` 时执行。
- 验证结论同步到 `HARDWARE_VALIDATION_CHECKLIST.md` 和 `ARCH_TASK_PROGRESS.md`。

## P3 - 数据质量、恢复和长期稳定性

- [ ] HR-P3-01 建立扫描质量标记：为位置超差、网分点数异常、幅相非有限值、保存重试和人工停止建立结构化质量字段。
- [ ] HR-P3-02 设计断点续扫：根据 scan session 和 trace index 识别已完整保存点，恢复时不得覆盖或重复计数，且必须重新验证硬件状态与坐标原点。
- [ ] HR-P3-03 增加硬件通信指标：记录各设备命令耗时、重试次数、超时次数、扫描时间和单点时间分布，支持定位偶发慢响应。
- [ ] HR-P3-04 建立长时间稳定性测试：执行规定点数或规定时长的循环扫描，统计位置误差、采样失败率、文件完整率和资源泄漏。
- [ ] HR-P3-05 收口硬件能力模型：端口数、频率范围、功率范围、最大点数和支持命令由连接后能力查询或型号 profile 提供，不再只依赖通用硬编码范围。

### P3 验收

- 每个扫描点具有可查询的数据质量状态。
- 异常退出后可安全恢复，且不会产生重复或错误位置数据。
- 日志可回答“哪台设备、哪条命令、耗时多久、重试几次、在哪个点失败”。
- 长时间测试达到项目约定的失败率和资源稳定性指标。
- E5080B/N5245B 的能力边界由 profile 或仪器查询驱动。

## 已完成基线

以下事项已完成，不再作为本 TODO 的未完成任务：

- VNA 连接配置支持 `E5080B` 和 `N5245B` 型号选择及 `*IDN?` 一致性校验。
