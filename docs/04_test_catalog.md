## 当前执行说明

### 天线校准

- 当前界面中的“校准类型”改为二选一：`发射校准` 固定走 `S21`，`接收校准` 固定走 `S12`，不再由用户手动选择测量参数。
- 软件启动时不再自动加载上一次天线校准结果；每次点击“开始校准”前，结果表格、结果文件路径和另存状态都会先清空，再开始新的校准流程。
- 校准执行过程中按通道实时刷新结果表格：每完成 1 个通道的 `H/V` 幅相采样，就立即在表格中追加 1 行，所有单元格内容居中显示。
- “结果另存”按钮移到“校准结果”区域，并且只有在本次校准成功生成结果文件后才可点击。
- 极化切换时序固定为：每次 `H/V` 切换成功后，固定等待 `20 ms` 再触发网分采样。
- 天线校准不走 `test_service.py` 和 `test_projects/common.py`，而是在主界面的独立“天线校准”选项卡内单独执行。
- 校准指令文件格式独立于频率/波位脚本，当前同时支持两种模板：
  1. 旧格式：三列 `类型 / 通道 / 指令`，`类型` 支持 `INIT`、`CLOSE_ALL`、`OPEN`、`CLOSE`，也兼容中文写法；`指令` 单元格可包含多行命令，运行时会逐行发送并逐条等待返回。
  2. 通道校准表格式：四列 `通道 / 校准指令 / 校准结束后发送指令 / 备注`，其中 `Init` 行作为初始化命令，`1..N` 行作为逐通道校准命令和该通道测完后的收尾命令，`Stop` 行作为全部通道校准完成后的结束命令；`csv/xls/xlsx` 都可直接读取。
- 校准页面单独配置：校准指令文件、指令间时延、校准频率、测量参数（`S21` / `S12`）、IF Bandwidth、网分功率。
- 校准时延规则：
  1. `Init` 和最终 `Stop/结束` 指令在收到返回后固定再等待 `1 s`。
  2. 通道切换过程中的命令（如打开通道、关闭通道、开始前的关闭所有通道）在收到返回后，再按界面配置的“指令间时延”继续。
- 开始校准后的执行顺序如下：
  1. 开关箱切到 `PASSIVE` 链路。
  2. 转台将 `Theta` 和 `Phi` 都移动到当前零位 `(0, 0)`。
  3. 网分配置单点测量参数：测量通道、频率、IF Bandwidth、输出功率，并切到单次触发保持模式。
  4. DUT 先执行初始化指令；若指令文件提供了“关闭所有通道”步骤，则在开始通道循环前执行一次。
  5. 按指令文件中的通道顺序逐个执行：发送当前通道的校准指令/打开指令 -> 切到 `H` 极化并测量一次 -> 切到 `V` 极化并测量一次 -> 发送当前通道的“校准结束后发送指令”/关闭指令。
  6. 所有通道完成后，若指令文件提供了 `Stop`，则执行 `Stop`；否则执行一次“关闭所有通道”作为收尾。
  7. 生成五列校准文件：`通道 / H幅度 / H相位 / V幅度 / V相位`，保存到 `software/output/antenna_calibration/`。
- 校准结果不画图，直接在校准页面的表格中展示，并支持“结果另存”。
- 当校准频率配置为多个值时，流程按频率顺序串行执行：先完成第 1 个频率的全部通道，再切到下一个频率；结果文件导出为 `xls` 工作簿，每个频率各占 1 个 sheet，校准页结果区也按频率使用独立选项卡展示，并实时显示总进度、已用时间和剩余时间。

### 方向图

- 支持 TX / RX 两条流程：TX 固定读取网分 `S21`，RX 固定读取网分 `S12`。
- 支持频率、波位、极化、Phi / Theta、IF Bandwidth、网分输出功率、转台速度档位配置。
- 测试项配置页新增“测试模式”选项，当前支持两种真实方向图执行顺序：
  1. `空间位置 - 频率/波位 - 极化`
  2. `波位 - 频率 - 空间位置 - 极化`
- 第二种模式用于优先减少 DUT 频率/波位重配置次数；外层先固定 `波位 + 频率`，再扫描全部转台空间位置，最后在每个位置依次完成 `H/V` 极化采样。
- 真实测试时联动转台、开关箱和网分；当前不连接 DUT 时，DUT 频率和波位配置先用 `10 ms` 延时占位。
- 网分不再使用 Segment 模式；真实测试时按 `频率数 x 波位数 x 极化数` 组织 measurement slot，但扫频基础参数只在进入流程时初始化一次；后续仅在频率变化时切换网分频率。
- 真实方向图测试期间，转台与网分优先保持 TCP 长连接，避免每条命令重复建连带来的额外耗时。
- 每个转台位置的真实方向图采样顺序如下：
  1. 开关箱在测试开始前切到 `PASSIVE` 链路。
  2. 转台回到当前零位，不执行完整复位。
  3. 网分配置测量参数：TX 使用 `S21`，RX 使用 `S12`；配置网分输出功率、IF Bandwidth，并保持 `SENS1:SWE:MODE HOLD`。measurement slot 顺序按 `频率 -> 波位 -> 极化` 组织，并在频率/波位维度上采用蛇形极化顺序，例如 `f1,b1,H -> f1,b1,V -> f1,b2,V -> f1,b2,H -> ...`。
  4. 转台移动到当前 `phi/theta` 位置并等待到位。
  5. 按当前位置的 slot 顺序逐项执行：切到当前 slot 的极化；每次 H/V 极化切换完成后固定等待 `3 ms` 再继续后续采样。
  6. DUT 频率/波位只在当前 slot 的 `频率或波位` 相对上一 slot 发生变化时才重新配置；当前无 DUT 时，用 `10 ms` 延时占位。
  7. 网分先通过 `configure_sweep(start=stop=首个频点, points=1)` 初始化 `LIN / points=1 / IFBW`；后续每个 slot 仅在频率变化时用 `set_frequency_point()` 切换到新频率，再执行一次 `SENS1:SWE:MODE SING;*OPC?`。
  8. 单次触发完成后，立即通过 `CALC1:DATA? SDATA` 读取当前 slot 的单点复数数据。
  9. 软件端根据该单点 `SDATA` 复数对同步解析出幅度与相位；相位不显示在曲线中，但写入原始 JSON/CSV。
  10. 同一 `频率 + 波位` 的 H/V 相邻 slot 直接复用当前 DUT 配置与网分频率；将当前 slot 返回的数据立即分发到对应曲线；当前位置全部 slot 采样完成后，才进入下一个 `theta` 位置；如果有多个 `phi`，继续采用蛇形连续扫描顺序减少回扫。
- 网分单次触发命令保持为 `SENS1:SWE:MODE SING;*OPC?`，且方向图流程中不要追加 `INIT1:IMM`，避免 E5080B 在当前扫频模式下上报 `init ignored`。
- 方向图测量流程中除 DUT 占位外不额外插入固定时延。
- 当测试模式为 `波位 - 频率 - 空间位置 - 极化` 时，真实执行顺序改为：
  1. 开关箱切到 `PASSIVE`，转台回当前零位，网分只初始化一次 `S21/S12 + IFBW + 功率 + 单点扫频`。
  2. 外层按 `波位 -> 频率` 组织 DUT/VNA 配置；同一 `波位 + 频率` 组合只在进入该组时配置一次 DUT，并在需要时切换一次网分频率。
  3. 对当前 `波位 + 频率` 组合，按配置的 `Phi / Theta` 空间路径完整扫描全部转台位置；相邻 `波位 + 频率` 组合之间按组做蛇形往返，上一组若以路径终点结束，下一组就从该终点反向扫回，避免每组都回到起点重新开始。
  4. 到达每个空间位置后，再按 `H -> V` 的顺序完成极化采样；极化切换后继续沿用既有固定等待时间。
  5. 当前 `波位 + 频率` 的全部空间位置完成后，再切到下一个频率；当前波位全部频率完成后，再切换到下一个波位。
- 结果显示按真实采样顺序实时更新；导出时支持原始数据、图像页和指标回填。
- 结果页“显示曲线”选项卡中，`极化 / 频率 / Phi / 波位` 等每个已出现的筛选维度都提供 `全选/全不选` 按钮，便于快速切换整组曲线显示。

### EIRP

- 当前统一为单个 `eirp` 测试项
- 通过 `DUT 类型` 区分天线板和整机
- 真实测试时联动开关箱、转台、网分，以及按配置选择的接收链路
- `EIRP 接收链路` 支持两种模式：
  - `频谱接收`：沿用当前 `网分发 + 频谱收`
  - `网分接收`：改为 `网分发 + 网分收`，测试开始前开关箱切到方向图链路 `PASSIVE`
- 导出时根据 DUT 类型写入 `天线板EIRP` 或 `整机EIRP`
- EIRP 结果曲线按 `H`、`V` 和 `Total` 三类组织：
  - `H` / `V` 为真实扫描得到的两条功率曲线
  - `Total` 为 `H` 与 `V` 的线性功率相加后再转换回 dB 的派生曲线
- EIRP 结果会读取 `resource/config_files/路损文件.xlsx` 中 `EIRP` sheet 的路损，并按频率/极化补偿；结果文件同时保留原始 `received_power_dbm` 与补偿后的 `eirp_dbm` 以供追溯。
- 结果页“测试过程与结果”中的 EIRP 功率扫描图，横轴显示 `网分输出功率 (dBm)`，纵轴显示 `EIRP (dBm)`；每条 `H / V / Total` 曲线都会根据估算出的 `1 dB` 压缩点高亮 `P1dB` 标记，便于直接观察压缩点。
- EIRP 结果页与离线分析页复用方向图的数据视图切换能力，支持 `补偿后数据 / 原始数据` 与 `线极化 / 圆极化` 两组显示模式；其中圆极化视图会派生 `LHCP / RHCP / Total` 曲线。
- EIRP 真实联机启动时只创建空曲线骨架，不再先执行模拟测试生成预览曲线或假 `raw_data`；界面曲线只由真实仪表采样进度回调逐点追加。
- 结果页“显示曲线”选项卡中，EIRP 同样支持对 `极化 / 频率 / 波位` 各维度执行 `全选/全不选`
- 当前软件中的 EIRP 联机流程按以下顺序执行：
  1. 根据 `EIRP 接收链路` 切换开关箱：
     `频谱接收` 时切到 `EIRP` 链路；`网分接收` 时切到方向图链路 `PASSIVE`。
  2. 转台仅控制 `Theta` 轴，转到当前波位对应的目标角度。
  3. DUT 切到当前 `theta` 对应波位；当前若未接 DUT，先用 `10 ms` 占位。
  4. 配置 DUT 频率和网分频率；当前若未接 DUT，DUT 频率配置同样用 `10 ms` 占位。
  5. 网分初始化为 `CW time` 单点频率输出模式，固定使用 `number of points = 1`、`sweep time = 1 s`；按界面配置设置起始功率、功率步进、终止功率和最大压缩点。
     `频谱接收` 时，频谱同步配置 `Span`、`RBW`、内部放大器等功率测量参数，界面输入单位为 `kHz`，下发时转换为 `Hz`；外部衰减默认 `0 dB`。
     `网分接收` 时，接收值读取 `S21` 幅度，并按 `网分输出功率 + S21(dB)` 换算等效接收功率。
  6. 对每个 `波位 + 频率`，先切到 `H` 极化并扫描完整 H 功率曲线；再切到 `V` 极化扫描完整 V 功率曲线；真实转台控制器会在 `set_polarization()` 内部完成 `20 ms` 定稳，EIRP 流程层不再重复追加一次固定等待；随后根据 H/V 两条已测曲线计算 `Total` 曲线。
  7. `频谱接收` 时，频谱 `Reference Level` 由软件自动调节：每个功率点触发并读取扫频最大值后，若测值高于当前 `Reference Level`，则按 `10 dB` 步进上调；若测值低于 `Reference Level - 10 dB`，则按 `10 dB` 步进下调；每次调整后立即重测当前点，直到测值落在 `[Reference Level - 10 dB, Reference Level]` 窗口内。
     `频谱接收` 时，网分每次完成功率配置后仍固定等待 `10 ms` 再继续接收端采样；`网分接收` 时不再额外插入这段固定等待，而是直接进入 `SENS1:SWE:MODE SING;*OPC?` 单次触发。
  8. 当网分到达终止功率，或曲线压缩量达到界面设置的最大压缩点后，停止当前扫功率，并根据当前曲线估算 `1 dB` 压缩点。
  9. 切换到下一个频率，重复步骤 4-8；当前波位下全部频率完成后，再切换到下一个波位，重复步骤 3-8。
  10. 全部波位与频率测试完成后，保存 `H / V / Total` 曲线、`1 dB` 压缩点摘要以及原始 JSON/CSV 数据，并在结果摘要/过程日志中记录 `EIRP 路损补偿` 状态。

### GT

- GT 结果页目标值为 `G/T(dB/K)`，当前参考 `resource/GT 测试流程说明(1).pdf`、`resource/GT N9020b 指令(1).pdf` 和 `resource/测试结果原始数据文件样例(1)/0001/0001GT*.csv` 组织流程。
- 当前软件已支持 GT 预览、真实联机执行、原始数据落盘以及 `0001_report.xls` 模板中的 GT 页导出。
- 核心计算式按资料使用：
  `G/T = G - N0 - 198.601 - Ld`
  其中：
  - `G` 为载波测量后换算得到的接收增益项
  - `N0` 为噪声功率谱密度测量值
  - `Ld` 为空损
- GT 测试开始前必须先准备空损表 `Ld`。资料给出的空损标定方式为：
  - 使用馈源、转台、标准喇叭天线和矢网搭建空损量测链路
  - 按 10.825 / 11.075 / 11.325 / 11.575 / 11.825 / 12.075 / 12.325 / 12.575 GHz 逐点量测
  - 依据 `矢网读数 - 蓝色线损 - 标准天线增益 = 空损` 生成各频点空损
  - 后续 GT 正式测试直接读取该空损表，不在 GT 流程中重新测量
- 建议的 GT 联机测试流程如下：
  1. 加载 GT 配置：频点列表、波位列表、极化信息、空损表、信号源功率和 DUT 配置。
  2. 开关箱切到 `GT` 链路。
  3. 转台转到目标 `theta` 角度；如有 `phi` 维度，按测试项配置执行。
  4. DUT 切到接收模式，并配置到对应频率和波位。
  5. 先做 `N0` 噪声测量：
     - 频谱切到 SA 模式：`:CONFigure:SANalyzer`
     - 打开 Noise Marker：`:CALCulate:MARKer1:FUNCtion NOISe`
     - 衰减设为 `0 dB`：`:SENSe:POWer:RF:ATTenuation 0`
     - Ref Level 设为 `-60 dBm`：`:DISPlay:WINDow:TRACe:Y:SCALe:RLEVel -60`
     - Marker Noise Span 设为 `1 kHz`：`:CALCulate:MARKer1:FUNCtion:BAND:SPAN 1000`
     - RBW 设为 `1000 Hz`：`:SENSe:BANDwidth:RESolution 1000`
     - VBW 设为 `10 Hz`：`:SENSe:BANDwidth:VIDeo 10`
     - Sweep Time 设为 `2 s`：`:SENSe:SWEep:TIME 2`
     - Sweep Points 设为 `1001`：`:SENSe:SWEep:POINts 1001`
     - Trace Type 设为 `WRITe`：`:TRACe1:TYPE WRITe`
     - Detector 设为 `NORMal`：`:SENSe:DETector:TRACe1 NORMal`
     - 记录 `N0`
  6. 再做 `Cr / G` 载波测量：
     - 关闭 Noise Marker：`:CALCulate:MARKer1:FUNCtion OFF`
     - 衰减设为 `10 dB`：`:SENSe:POWer:RF:ATTenuation 10`
     - Ref Level 设为 `0 dBm`：`:DISPlay:WINDow:TRACe:Y:SCALe:RLEVel 0`
     - Span 设为 `1 MHz`：`:SENSe:FREQuency:SPAN 1000000`
     - RBW 设为 `9000 Hz`：`:SENSe:BANDwidth:RESolution 9000`
     - VBW 设为 `10000 Hz`：`:SENSe:BANDwidth:VIDeo 10000`
     - Sweep Time 设为自动：`:SENSe:SWEep:TIME:AUTO 1`
     - Sweep Points 设为 `1001`：`:SENSe:SWEep:POINts 1001`
     - Trace Type 继续使用 `WRITe`：`:TRACe1:TYPE WRITe`
     - Detector 设为 `NORMal`：`:SENSe:DETector:TRACe1 NORMal`
     - 打开馈源信号，记录载波测量结果以及原始功率读数
  7. 按当前链路修正关系计算 `G`，并保留原始字段 `P1`、`P2`、`SG_Power`、`Ld_H`、`Ld_V`。
  8. 按 `G/T = G - N0 - 198.601 - Ld` 计算当前频点、当前角度的 GT。
  9. 切换下一个频点，重复步骤 4-8；同一角度测完全部频点后切换到下一个角度。
  10. 全部角度完成后生成汇总结果和原始结果文件。
- 结果文件建议保持与现有样例一致：
  - `0001GT.csv`
    保存 `Theta x Frequency` 汇总 GT 值
  - `0001GT_all.csv`
    保存逐点原始数据，字段至少包括：
    `Theta(deg), Freq (MHz), P1, P2, G, N0, SG_Power, Ld_H, Ld_V, G/T`
- 当前对 N9020B 指令的理解与约束：
  - 文档中 `preamp`、`sweep.count`、`trace.average` 没有明确对应的 N9020B 指令，当前实现应先按“未使用/保持默认”处理
  - GT 测试优先保证 `N0` 和 `Cr` 两套频谱配置可重复切换，并完整保存中间量，避免只保留最终 `G/T`
  - 若后续联机时发现 `G` 的计算链路还包含固定增益、线损或极化修正，应继续保留 `G` 前的原始测量值，计算规则单独落到实现代码与文档中

## DUT 指令文件

- 有源 DUT 现在支持通过界面选择矩阵模板指令文件驱动控制，参考 `resource/dut_command_template.csv`。
- 指令文件支持 `csv`、`txt`、`tsv`、`xls`、`xlsx`，当前主模板格式如下：
  - 第 1 行：第 1 列为 `Init`，第 2 列为 `Stop`，第 3 列开始为频率表头。
  - 第 2 行：第 1 列为初始化指令列表，第 2 列为结束指令列表，第 3 列开始为各频率对应的波位。
  - 第 3 行到第 66 行：第 3 列开始的每一列都是当前“频率 + 波位”配置的指令列表，典型为连续 `64` 条指令。
- 测试流程命中当前频率和波位后，会读取该列全部非空单元格并按顺序发送：
  - 第一次配置该频率/波位时，整列指令全部发送。
  - 后续再次切换时，软件会按“指令关键字 + 芯片地址/寄存器地址”维护运行时暂存区，只重发内容发生变化的那几条指令；未变化指令直接跳过。
- 每条指令都必须收到返回才算成功；单条指令默认超时 `1 s`。如果同一配置下需要连续发送多条指令，则当前一条收到返回后立即发送下一条，不额外插入固定时延。
- `Init` 和 `Stop` 属于整轮测试级别的特殊指令：测试开始前执行一次 `Init`，测试结束后执行一次 `Stop`；执行完成后仍按测试流程固定等待 `5 s`，保证 DUT 状态稳定。
- 方向图、GT、EIRP 的真实联机流程中，只要检测到“有源 DUT + 已配置指令文件”，就会优先使用该文件完成 DUT 配置；未配置时保持原有流程。

## 2026-04-08 补充

### TX / RX 方向图与有源 DUT 指令文件

- 有源 DUT 在方向图联机测试中，必须先配置 `指令文件`；无源 DUT 不执行该流程。
- 指令文件按 `频率 / 波位 / 指令列表` 匹配；若当前频率和波位没有匹配到指令，方向图测试直接报错并停止该测试项。
- 支持特殊首列 `init/start/initialize` 和 `stop/end/close/final`：
  - 测试开始前先执行一次 `init`，随后固定等待 `5 s`
  - 测试结束后执行一次 `stop`，随后固定等待 `5 s`
- 频率/波位配置采用“差异发送”策略：
  - 软件在运行时维护一份 DUT 指令暂存区
  - 每次切换到新的频率/波位时，只发送与当前暂存状态不同的指令行
  - 完全相同的指令不重复发送，用于减少测试耗时

### 方向图结果保存

- 方向图原始结果仍保留原始 `H/V` 测量曲线与相位数据。
- 同时新增四组派生结果视图并写入原始 JSON/CSV：
  - `raw_linear`
  - `final_linear`
  - `raw_circular`
  - `final_circular`
- `final_*` 视图使用系统配置中的校准文件，对 H/V 幅度与相位进行补偿后生成。
- 圆极化结果按 H/V 幅相计算 `LHCP / RHCP / Total`。
- 原始数据文件中会同时保存本次使用的校准来源文件及对应的 H/V 校准幅相值，便于追溯。

### 测试结果计算与补偿

- 线极化 `Total` 的计算方式：
  - 对同一角度点的 `H`、`V` 幅度先转为线性功率：`P = 10^(dB/10)`。
  - 再做功率相加：`P_total = P_H + P_V`。
  - 最后转回 dB：`Total(dB) = 10 * log10(P_total)`。
- 方向图的“补偿后数据”来源于系统配置中选择的校准文件：
  - `H_final_amp = H_raw_amp - H_cal_amp`
  - `V_final_amp = V_raw_amp - V_cal_amp`
  - `H_final_phase = H_raw_phase - H_cal_phase`
  - `V_final_phase = V_raw_phase - V_cal_phase`
  - 校准文件按频率优先匹配；若没有完全相同频率，则使用最近频率；通道优先按波位编号匹配，缺失时再按当前波位序号回退。
- 圆极化结果基于 H/V 幅相恢复复电场后计算：
  - `E_H = 10^(H_amp/20) * exp(j * phi_H)`
  - `E_V = 10^(V_amp/20) * exp(j * phi_V)`
  - `LHCP = (E_H + jE_V) / sqrt(2)`
  - `RHCP = (E_H - jE_V) / sqrt(2)`
  - `LHCP/RHCP` 曲线显示值使用 `20 * log10(|E|)`。
  - 圆极化 `Total` 仍按总功率显示：`10 * log10(|E_H|^2 + |E_V|^2)`。
- 导出报表时，方向图类结果的关键指标按补偿后曲线计算：
  - `天线板EIRP下降(dB)`：参考峰值减当前曲线峰值。
  - `波束宽度(dB)`：按 `3 dB` 波束宽度计算。
  - `指向精度(°)`：主瓣峰值角度减目标角度。
  - `轴比(dB)`：使用同角度点的补偿后 H/V 幅相按极化椭圆公式计算。
  - `第一旁瓣抑制(dBc)`：主瓣峰值与最大旁瓣的差值。
- EIRP 结果中的 `Total` 与方向图一致，按 `H`、`V` 两条功率曲线先转线性再相加，最后转回 dB。
- GT/增益相关结果中的线损补充逻辑如下：
  - 软件从校准/空损文件中读取 `Ld_H`、`Ld_V`。
  - 若当前频率没有完全匹配项，则回退到最近频率的线损值。
  - GT 计算继续使用 `G/T = G - N0 - 198.601 - Ld`，其中 `Ld` 取当前极化链路对应的线损；原始结果中同时保留 `Ld_H`、`Ld_V` 便于追溯。

## 2026-04-14 补充

### 系统配置与结果分析

- 开关箱通信检查只有在响应内容包含 `DONE` 时才判定成功；有回包但未包含 `DONE` 仍视为失败。
- 设备切换到 `Serial` 接口时，如果原配置不是有效 `COM` 口，则不再自动占用首个检测到的串口，也不会把 TCP 端口号错误带入波特率输入框。
- “加载数据”按钮在离线导入前，必须先执行一次与“清空结果”按钮一致的结果清理逻辑，避免新旧结果混在同一批次视图中。

### 天线校准前置校验

- 真实联机校准在打开设备会话前，先检查开关箱、转台、网分、DUT 是否复用了同一串口；如果存在冲突，直接报配置错误并停止流程。
- 开关箱通信异常应在开关箱控制阶段直接报出，不再以 DUT 建链失败的形式误导定位。
